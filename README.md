Bu proje, uçtan uca otomatik bir web veri çekme (scraping) ve görselleştirme hattıdır. Bir film izleme sitesinden film verilerini (Film Adı, IMDb Puanı, FHD Puanı, Çıkış Yılı ve Türü) dinamik olarak çeker, elde edilen ham veriyi temizler ve interaktif, canlı bir web arayüzü üzerinden kullanıcıya sunar.

Bu platform; ölçeklenebilir otomasyon pratiklerini, dinamik DOM (Sayfa Yapısı) yönetimini ve veri yapılandırma yeteneklerini sergilemek amacıyla geliştirilmiştir.

## 🎯 QA ve SDET Vurguları (Nasıl Geliştirdim?)
Kalite güvencesi (QA) ve hatasız otomasyon mimarilerine odaklanan bir geliştirici olarak, bu projede çeşitli dayanıklı (resilient) stratejiler uyguladım:
* **Dinamik Bekleme Stratejileri:** Koda sabitlenmiş (hard-coded) kör beklemeler olan `time.sleep()` yerine, elementlerin tam olarak yüklendiğinden emin olmak ve veri çekme hızını maksimuma çıkarmak için Selenium'un `WebDriverWait` (Akıllı/Açık Beklemeler) özelliğini kullandım.
* **DOM Tutarsızlıklarının Yönetimi:** Veri çekme mantığını, değişken HTML yapılarını idare edebilecek şekilde güncelledim (örneğin; çıkış yılını, sitede "Yıl" veya "Yapım" gibi farklı etiketlerle yer alsa bile tespit edip çekebilecek bir yapı kurdum).
* **Veri Temizleme (Sanitization):** Çekilen karmaşık metinleri Pandas ve Regex kullanarak işledim; hassas arayüz filtrelemesi yapabilmek için metinleri doğru sayısal verilere (float/int) dönüştürdüm.
* **Sürekli Dağıtım (CI/CD Yaklaşımı):** Kesintisiz güncellemeler ve projenin canlıda kalması için GitHub deposunu Streamlit Cloud sunucularına bağladım.

# 🎬 Movie Scraper & Interactive Filtering Platform (Film Robotum)

## 📌 About The Project
This project is an end-to-end automated web scraping and data visualization pipeline. It dynamically extracts movie data (Titles, IMDb Ratings, FHD Ratings, Release Years, and Genres) from a movie streaming website, cleans the structured data, and presents it through an interactive, live web interface.

This project was built to demonstrate scalable automation practices, dynamic DOM handling, and data structuring capabilities.

## 🚀 Live Demo
You can try the live application here: **[Movie Filtering Platform](https://fullhd-film-robotu.streamlit.app/)**

## 🛠️ Tech Stack & Tools
* **Data Extraction / Automation:** Python, Selenium WebDriver, BeautifulSoup (bs4)
* **Data Processing & Cleaning:** Pandas, Regex (re)
* **Front-End / Deployment:** Streamlit, Streamlit Community Cloud

## 🎯 QA & SDET Highlights (How I Built It)
As a developer focused on quality assurance and robust automation, I implemented several resilient strategies in this project:
* **Dynamic Waiting Strategies:** Instead of hard-coded `time.sleep()`, the scraper utilizes Selenium's `WebDriverWait` (Explicit Waits) to ensure elements are fully loaded, preventing flaky extractions and optimizing the scraping speed.
* **DOM Inconsistency Handling:** Adapted the scraping logic to handle varying HTML structures (e.g., extracting the release year dynamically even when the site uses different labels like "Yıl" or "Yapım").
* **Data Sanitization:** Processed raw strings using Pandas and Regex to clean and convert unstructured text into accurate numerical data (float/int) for precise UI filtering.
* **Continuous Deployment:** Connected the GitHub repository to Streamlit Cloud for seamless updates and live hosting.

## 📂 Project Structure
* `FULLHD_VERİ_ÇEKME.py`: The core automation script. Navigates through pages, waits for specific elements, parses HTML, and exports data.
* `arayuz.py`: The front-end Streamlit application providing interactive sliders and search functionality.
* `filmler.csv`: The structured database generated by the scraper.
* `requirements.txt`: Dependencies for the cloud deployment.
